## 6. Procédure opérationnelle

### ÉTAPE 0 — Backup obligatoire (avant toute suppression)
```bash
# Sur le serveur, depuis le dossier MySQL ou via SSH
mysqldump -u [user] -p[pass] prestanew > backup_prestanew_$(date +%Y%m%d_%H%M).sql
gzip backup_prestanew_*.sql
```
> ⚠️ **Ne pas sauter cette étape.** La suppression définitive est irréversible.

### ÉTAPE 1 — Table rase (DELETE en cascade)
```sql
-- Désactiver les contraintes FK temporairement
SET FOREIGN_KEY_CHECKS = 0;

-- Supprimer les combinaisons / attributs
DELETE FROM ps_product_attribute;
DELETE FROM ps_product_attribute_combination;
DELETE FROM ps_product_attribute_image;

-- Supprimer les langues et shop entries
DELETE FROM ps_product_lang;
DELETE FROM ps_product_shop;

-- Supprimer les catégories associées
DELETE FROM ps_category_product;

-- Supprimer les images
DELETE FROM ps_image;
DELETE FROM ps_image_lang;
DELETE FROM ps_image_shop;

-- Supprimer les produits
DELETE FROM ps_product;

-- Réinitialiser les auto-increment
ALTER TABLE ps_product AUTO_INCREMENT = 1;

-- Réactiver les FK
SET FOREIGN_KEY_CHECKS = 1;
```
> Après le DELETE, vider le cache PS : Admin → Paramètres avancés → Performance → Vider le cache.
